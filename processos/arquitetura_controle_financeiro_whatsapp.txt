Preciso implementar no bot de WhatsApp um módulo completo de controle financeiro conversacional.

OBJETIVO
Permitir que o usuário registre despesas, receitas e transferências por linguagem natural, usando texto inicialmente, com possibilidade de expansão para áudio, imagem, comprovantes e PDFs. O sistema deve interpretar a mensagem, extrair dados estruturados, validar, salvar em banco e responder no próprio WhatsApp.

ARQUITETURA ESPERADA
1. Camada de recepção via webhook do WhatsApp.
2. Camada de orquestração/backend para processar mensagens.
3. Camada de IA para identificar intenção e extrair entidades financeiras.
4. Camada de regras de negócio para validar, pedir confirmação quando necessário e persistir os dados.
5. Camada de persistência em banco relacional.
6. Camada de relatórios, orçamento, recorrência e lembretes.

FUNCIONALIDADES OBRIGATÓRIAS DO MVP
- Registrar despesa por texto.
- Registrar receita por texto.
- Consultar total gasto hoje, na semana e no mês.
- Consultar total por categoria.
- Editar o último lançamento.
- Excluir o último lançamento.
- Manter estado conversacional para perguntas complementares quando faltarem dados.
- Responder de forma curta e objetiva no WhatsApp.

REGRAS DE NEGÓCIO
- Se confiança alta e sem campos faltantes, salvar automaticamente.
- Se houver ambiguidade, pedir confirmação.
- Se faltar campo crítico, abrir fluxo guiado.
- Usar categorias controladas, sem permitir que a IA invente categorias livres.
- Tratar duplicidade provável de lançamentos.
- Converter datas relativas para datas absolutas.
- Manter a base como fonte de verdade, nunca a conversa.

FLUXO PRINCIPAL
1. Usuário envia “gastei 50 no mercado”.
2. Sistema interpreta como despesa de R$ 50 em mercado na data atual.
3. Regras validam.
4. Sistema salva.
5. Bot responde “Salvei R$ 50,00 em Mercado.”

FLUXO COM AMBIGUIDADE
1. Usuário envia “paguei 120”.
2. Sistema identifica despesa de R$ 120, mas sem categoria.
3. Bot pergunta “Foi com o quê?”
4. Usuário responde.
5. Sistema completa o draft e salva.

OBSERVAÇÕES IMPORTANTES
- A fonte de verdade deve ser o banco de dados, não a memória da conversa.
- A IA deve interpretar e estruturar, mas a decisão final é do backend.
- O sistema deve ser tolerante a linguagem informal.
- O sistema deve priorizar baixa fricção e respostas rápidas.
- O processamento de mídia deve ser assíncrono.
- O projeto deve ser iniciado pelo núcleo financeiro e depois expandido para multimodalidade e automações.
